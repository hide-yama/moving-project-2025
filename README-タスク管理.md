# タスク管理の使い方

このプロジェクトでは、シェルスクリプトを使って未完了タスクを自動抽出します。

---

## 📁 ファイル構成

- **`タスク一覧.md`** - マスターのタスク一覧（手動編集）
- **`未完了タスク一覧.md`** - 未完了タスクのみを抽出したファイル（自動生成、編集禁止）
- **`generate_uncompleted_tasks.sh`** - 未完了タスクを抽出するスクリプト

---

## 🔄 ワークフロー

### 1. タスクを完了した時

タスク一覧.mdで該当タスクのチェックボックスを更新：

```markdown
- [ ] タスク名  →  - [x] タスク名
```

### 2. 未完了タスク一覧を更新

ターミナルでスクリプトを実行：

```bash
cd "/Users/yamamotohideki/プロジェクト管理/引越し計画"
./generate_uncompleted_tasks.sh
```

または、プロジェクトフォルダ内で：

```bash
./generate_uncompleted_tasks.sh
```

### 3. 未完了タスク一覧を確認

`未完了タスク一覧.md` を開いて、残りのタスクを確認します。

---

## 📊 スクリプトの出力例

```
✅ 未完了タスク一覧を生成しました: 未完了タスク一覧.md

📊 統計情報:
  総タスク数: 74
  完了: 34
  未完了: 40
  進捗率: 45%
```

---

## ⚠️ 重要な注意事項

1. **`未完了タスク一覧.md` は自動生成ファイルです**
   - このファイルを直接編集しないでください
   - 編集してもスクリプト実行時に上書きされます

2. **タスクの編集は `タスク一覧.md` で行ってください**
   - タスクの追加、完了マーク、内容変更は全てこちらで行います

3. **スクリプトは何度でも実行可能**
   - タスクを完了するたびに実行して、最新の未完了タスクを確認できます

---

## 🎯 活用例

### タスク完了後の流れ

1. タスク一覧.mdでタスクを完了マーク（`- [x]`）
2. スクリプトを実行
3. 未完了タスク一覧.mdで残りのタスクを確認
4. 次に何をすべきか把握

### 毎日の確認ルーティン

朝や夜に `未完了タスク一覧.md` を見て、今日やるべきタスクを確認する習慣をつけると効率的です。

---

## 🔧 トラブルシューティング

### スクリプトが実行できない

実行権限がない場合は、以下のコマンドで権限を付与：

```bash
chmod +x generate_uncompleted_tasks.sh
```

### スクリプトの場所がわからない

プロジェクトフォルダに移動：

```bash
cd "/Users/yamamotohideki/プロジェクト管理/引越し計画"
```

### 統計情報が表示されない

スクリプトが正常に実行されていれば、必ず統計情報が表示されます。
エラーメッセージが出ている場合は、タスク一覧.mdのフォーマットを確認してください。

---

## 📝 スクリプトの仕組み

スクリプトは以下のロジックで動作します：

1. **セクション単位で解析**
   - `##` や `###` で始まるセクションヘッダーを認識

2. **未完了タスクの検出**
   - `- [ ]` で始まる行を未完了タスクとして検出
   - その配下の子タスク（完了・未完了問わず）も含めて抽出

3. **完了タスクの除外**
   - `- [x]` で始まるタスクとその配下は全て除外

4. **文脈の保持**
   - セクションヘッダーや説明文も適切に含めて出力
