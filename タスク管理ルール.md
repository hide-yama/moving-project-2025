# タスク管理ルール

最終更新: 2025年10月31日

---

## ✅ タスクのチェックルール

### 基本原則

**子タスクがある親タスクは、すべての子タスクが完了するまで完了にしない**

### 具体例

#### ❌ NG例（ルール違反）
```markdown
- [x] 公共料金の解約手続き（親タスク完了）
  - [x] 電気の停止手続き
  - [x] ガスの停止手続き
  - [ ] 水道の停止手続き ← まだ未完了なのに親タスクが完了している
```

#### ✅ OK例（ルール準拠）
```markdown
- [ ] 公共料金の解約手続き（親タスク未完了）
  - [x] 電気の停止手続き
  - [x] ガスの停止手続き
  - [ ] 水道の停止手続き ← 未完了の子タスクがあるため親タスクも未完了
```

すべての子タスクが完了したら、親タスクにチェック：
```markdown
- [x] 公共料金の解約手続き - 完了: 10/30
  - [x] 電気の停止手続き - 完了: 10/30
  - [x] ガスの停止手続き - 完了: 10/30
  - [x] 水道の停止手続き - 完了: 10/30
```

## 📝 タスクのチェック手順

### 子タスクがない場合

タスクが完了したら即座にチェック：
```markdown
- [ ] 住民票の取得 → - [x] 住民票の取得 - 完了: 10/30
```

### 子タスクがある場合

1. **子タスクを1つ完了**
   ```markdown
   - [ ] 公共料金の解約手続き
     - [x] 電気の停止手続き - 完了: 10/30
     - [ ] ガスの停止手続き
     - [ ] 水道の停止手続き
   ```

2. **すべての子タスクが完了したら親タスクにチェック**
   ```markdown
   - [x] 公共料金の解約手続き - 完了: 10/30
     - [x] 電気の停止手続き - 完了: 10/30
     - [x] ガスの停止手続き - 完了: 10/30
     - [x] 水道の停止手続き - 完了: 10/30
   ```

3. **未完了タスク一覧を更新**
   ```bash
   ./generate_uncompleted_tasks.sh
   ```

---

## 🔍 親タスクと子タスクの見分け方

### 親タスク
- インデントなし（行頭から開始）
- `- [ ]` または `- [x]` で始まる
- 配下に子タスクを持つ

### 子タスク
- インデントあり（スペース2個以上）
- `  - [ ]` または `  - [x]` で始まる
- 親タスクに属する

---

## ⚠️ 注意事項

### 子タスクのみ完了して親タスクを放置しない

子タスクがすべて完了したら、必ず親タスクにもチェックを入れる。

### 親タスクを先に完了させない

子タスクがまだ残っているのに、親タスクだけ完了にしない。

### 定期的に整合性チェック

時々タスク一覧全体を見直して、親子関係のチェック状態が矛盾していないか確認する。

---

## 🎯 このルールのメリット

1. **進捗が正確に把握できる**
   - 親タスクの完了率 = 実際の作業進捗率

2. **未完了タスク一覧が信頼できる**
   - 親タスクが未完了なら、その配下に必ず未完了の子タスクがある

3. **タスク漏れを防止**
   - 親タスクを完了させる際に、子タスクを全部チェックする習慣がつく
